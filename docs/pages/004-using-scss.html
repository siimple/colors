All .scss source files that powers siimple colors are placed in ./scss folder. There are some details about the files that you can import from this folder:
File
Description
palette.scss
Base colors definition
colors.scss
Where the variables for each color are defined
functions.scss
Utility functions to work with colors
Adding siimple-colors to your SCSS files
You can copy the scss folder into your project and import the file that you need with the @import rule using relative paths, or you can add the parent folder where siimple colors is located to the includePaths option of the SASS compiler.
For example, if you have installed siimple colors using a package manager (like npm or yarn), you should add the node_modules folder to the includePaths option of your SASS compiler. 
$ node-sass --include-path ./node_modules source.scss output.css
And then you can import the SCSS files adding the following lines to your .scss file: 
@import "@siimple/colors/scss/palette.scss";
@import "@siimple/colors/scss/colors.scss";
@import "@siimple/colors/scss/functions.scss";
Using SCSS variables
You can include the colors in yout SCSs files using the variables with the structure $siimple-{COLOR}-{LIGHTNESS}. You should import at least colors.scss to use these variables. 
.my-style {
    background-color: $siimple-grey-light;
    color: $siimple-blue;
}
The base color has no lightness suffix, so you should use $siimple-{COLOR} instead of $siimple-{COLOR}-base.
Using SCSS functions
You can also load colors using the SCSS function siimple-color($color, $lightness). The first argument is a string with the color name, and the second argument (optionally) is a string with the lightness of the color (extra-dark, dark, base, light or extra-light). You can omit the second argument if you want the base color. 
.my-style {
    background-color: siimple-color("grey", "light");
    color: siimple-color("blue");
}
